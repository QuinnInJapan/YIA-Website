# YIA 新ウェブサイト — プレゼン資料

---

## 1. yia.jp は変わりません

今お使いの「yia.jp」というアドレスは、そのまま変わりません。

### 変わらないこと

- **yia.jp** のアドレスはそのまま使えます
- **So-net**（プロバイダー）も変わりません — 引き続きYIAが管理します
- **メールアドレス**やブックマークへの影響はありません
- 垣谷さん（ブルーダイヤモンド）との関係もこれまで通り維持できます

### 古瀬さんにご確認いただいた3点

| 確認事項               | 結果                 |
| ---------------------- | -------------------- |
| So-netのアカウント情報 | ✅ 確認済み          |
| ドメイン更新料の支払い | ✅ YIAが直接支払い中 |
| 業者との契約内容       | ✅ 問題なし          |

### 切り替え作業

So-netの管理画面で**設定を1つ変更するだけ**です（数分で完了）。それだけで、yia.jp にアクセスすると新しいサイトが表示されるようになります。

## 2. 新しいサイトの紹介

### トップページ

新しいトップページには、以下の要素が含まれています：

- **ヒーロー画像** — サイトの顔となる大きな写真
- **お知らせ一覧** — 最新のお知らせを表示
- **活動紹介カード** — YIAの主な活動をカード形式で紹介
- **YIAについて** — 協会の紹介文
- **活動写真** — 活動の様子を写真で紹介（6枚）
- **アクセス情報** — 住所・地図・連絡先

### カテゴリページ（5つ）

サイトの内容は5つのカテゴリに分かれています：

1. **相談・サービス**
2. **教室・講座**
3. **イベント**
4. **交流・協力**
5. **協会について**

### 各ページの特徴

- テキスト、画像、スケジュール、料金表、ギャラリーなど**柔軟に構成**
- 内容に合わせて最適な表示方法を選んでいます

### お知らせページ

- **一覧表示** — 新しい順に表示
- **ピン留め機能** — 重要なお知らせをページ上部に固定表示
- **詳細ページ** — 各お知らせの全文を表示

### お問い合わせ

- **フォーム付きページ** — 訪問者が直接メッセージを送れます

### その他

- **日本語・英語の両方に対応**（バイリンガル）
- **スマートフォン対応** — どの端末からでも見やすいレスポンシブデザイン

## 3. 自分たちで更新できること

管理画面（Studio）を使って、YIAのスタッフが**直接**以下の操作ができます。ブラウザからアクセスでき、特別なソフトは不要です。

### お知らせの作成・編集・削除

- タイトル、日時、本文（日本語・英語）
- 画像の添付
- PDFなどのファイル添付
- **重要なお知らせをトップページに固定表示**（ピン留め）

### ホームページの更新

- ヒーロー画像の差し替え
- 表示するお知らせの選択
- 活動写真（6枚）の入れ替え
- イベントチラシの追加

### 各ページの内容編集

- テキスト、画像の変更
- スケジュール、料金表の更新
- PDF・ドキュメントの添付

### 画像の管理

- 新しい画像のアップロード
- トリミング・表示位置の調整（切り抜きツール付き）

### サイト基本情報の更新

- 電話番号、住所、営業時間
- Google Maps の表示
- その他の連絡先情報

## 4. 変更できないこと（開発者が対応する部分）

以下の変更が必要な場合は、Quinn にご相談ください：

- サイト全体の**デザイン**（色、フォント、レイアウト）
- ホームページの**構成**（セクションの順番や種類）
- **ナビゲーション**（メニュー）の大幅な変更
- 新しい種類の**ページやセクション**の追加
- **新機能**の追加（例：会員ログイン、オンライン予約など）

### ただし

**管理画面（Studio）はカスタマイズ可能です。** たとえば：

- 「この項目がわかりにくい」→ 名前や説明文を変更できます
- 「ここにこういう入力欄がほしい」→ 追加できます
- 「この操作が面倒」→ もっと簡単な方法に変えられます

**使いながら気づいたことがあれば、遠慮なくお知らせください。随時改善していきます。**

## 5. 費用について

### 現在の費用

| 項目                         | 費用       | 備考                                    |
| ---------------------------- | ---------- | --------------------------------------- |
| **ドメイン（yia.jp）**       | 年間数千円 | すでにYIAがSo-netに支払い中（変更なし） |
| **ホスティング（Vercel）**   | **無料**   | サイトを公開するためのサービス          |
| **コンテンツ管理（Sanity）** | **無料**   | 管理画面を提供するサービス              |

### ホスティング（Vercel）について

- 現在は**無料プラン**を使用しています
- YIAのような小規模サイトでは十分な容量です
- アクセス数が大幅に増えた場合は、月額約$20（約3,000円）のプランに変更する可能性がありますが、**当面は不要**です

### コンテンツ管理（Sanity）について

- 現在は**無料プラン**を使用しています
- 無料枠の内容：
  - 月50万回のAPIリクエスト（ページ表示のたびに使用）
  - 20GBのアセット容量（画像・PDFなどの保存容量）
- YIAの規模では**十分余裕**があります
- 超えた場合は月額$15程度〜（約2,200円〜）ですが、**当面は不要**です

## 6. 今後の流れ

### 現在

- 各ページの内容を確認・調整中です
- 新年度（4月〜）に合わせて変更が多いため、以下の流れで進めています：
  1. **Quinn が初稿を作成**
  2. **YIA に内容を確認**いただく
  3. 修正があれば対応

### 次のステップ

- **管理画面（Studio）の使い方をご説明**します
- 実際に操作していただきながら覚えていただきます

### 公開（切り替え作業）

- 準備ができたら、So-net で **DNS設定を変更**
- それだけで yia.jp に新しいサイトが表示されます

### 切り替え時に起こりうること

ウェブサイトの「住所の案内先」を変更する作業なので、切り替え直後に一時的な問題が発生する可能性があります。**いずれも想定内であり、対応できます**が、念のためご説明します。

| 起こりうること                       | 説明                                                                                                                                                                | 対応                             |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------- |
| **サイトが一時的に表示されない**     | DNS設定の変更がインターネット全体に反映されるまで、数分〜数時間かかることがあります。その間、一部の方には旧サイトが、別の方には新サイトが表示される場合があります。 | 待つだけで自然に解消します       |
| **「安全でないサイト」と表示される** | 新しいサーバーのセキュリティ証明書（SSL）の発行に数分かかります。その間、ブラウザが警告を出す場合があります。                                                       | 通常は数分で自動的に解消します   |
| **管理画面（Studio）が使えない**     | 管理画面は許可されたURLからのみアクセスできるように設定されています。yia.jp に切り替えた後、新しいURLを許可リストに追加する必要があります。                         | Quinn が設定を変更して対応します |

### 切り替えのタイミング

上記の理由から、切り替え作業は**時間外（営業時間外や週末）**に行うことをおすすめします。

- 万が一問題が起きても、サイトの訪問者への影響が最も少ない時間帯
- Quinn が切り替え後すぐに動作確認・問題対応できる時間帯
- **事前にテストはしますが、「本番環境での確認」は切り替え後にしかできないものもあります**

### 公開後

- しばらくの間、**一緒に確認・調整**を行います
- 問題がないか、表示に不具合がないかを確認します
